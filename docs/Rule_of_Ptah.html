The Rule of Ptah

Protocols for Ibis Labs LLC // Scribe: PETE // Agent: KA

Preamble: We honor Ptah (Craftsmanship) and Ma'at (Order) in all development. We build with intent, preserve what is sacred, and seek balance between creativity and structure.

THE BACKEND DOES NOT EXIST FOR ITS OWN GLORY. IT EXISTS SOLELY TO SUPPORT THE USER IN THEIR ETERNAL STRUGGLE TO WRESTLE MEANING FROM THE CHAOS OF EXISTENCE.

THE SYSTEM MUST NEVER PUNISH THE USER FOR BEING HUMAN.

WE CONVERT THE ANXIETY OF CHAOS (NUN) INTO THE PEACE OF ORDER (MA'AT).

  1. Part I: The Prime Directive (Team Work)
    1. Principle: "When we collaborate, we collaborate."
      • When the goal is unclear, we seek clarification.
      • We document with TypeScript, clear comments in the code, and detailed commit messages.
      • We review the code AND the development environment output before signing off on any project no matter how large or small.
      • All project team members assigned a role on a branch must sign off on changes, fixes, features, or improvements BEFORE pushing code to a remote repository.
  2. Part II: The Lotus Directive (Mobile First)
    1. Principle: "The Scribe shall build for the palm before the desktop."
      • All design and development shall prioritize the mobile user experience as the bedrock of creation.
      • We seek to bring Ma'at (Order) to the user by delivering an unparalleled, holistic experience on the small screen.
      • The glory of a feature is measured by its seamless function and aesthetic on a mobile
